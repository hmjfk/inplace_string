# 容量固定の文字列型「inplace_string」
## はじめに
C++26では、`std::vector`の積み上げ領域版`std::inplace_vector`が追加される。ということは、独立環境でも動的配列が使えるようになるのである。

ここで、新たな疑問が生じる。`std::vector`と同じ記憶域戦略を持っていた`std::string`を独立環境でも使えるようにすることはできないのか？
一応、C++17で`std::string_view`は追加されたものの、`std::string`とは違って文字列の連結はできない。

そこで、文字列の連結ができる`std::string_view`を`inplace_string`と名付け、実装してみることにした。この場所は、その成果を部品集としてまとめたものである。

## 使い方
先行宣言列のみで構成される。単に`#include <inplace_string>`とすれば使用できる。
